# Phase 6 ‚Äî Localization Migration (gen-l10n + diacritics) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the hand-rolled 1,348-line `AppLocalizations` string-map class with Flutter's standard `gen-l10n` (ARB files + compile-time-safe generated accessors + ICU plurals), restore Romanian/Hungarian diacritics everywhere (UI, Cloud Function notifications, Android channel names), embed a Unicode font so journal PDFs render diacritics, and make all dates locale-aware.

**Architecture:** Create `lib/l10n/app_ro.arb`, `app_hu.arb`, `app_en.arb` (RO as template) and a `l10n.yaml`; `flutter gen-l10n` produces `AppLocalizations` with the same call surface (`l10n.someKey`). Migrate call sites from the custom class to the generated one (mostly identical). Delete the custom class. Convert hand-coded plurals (`teamsCount`, `relativeTime`, etc.) to ICU messages. Embed Noto Sans in the PDF service. Pass `Localizations.localeOf(context)` to every `DateFormat`.

**Tech Stack:** Flutter `flutter_localizations` + `gen_l10n` (built in), `intl`, `pdf` with a bundled TTF asset.

**Branch:** `phase6-l10n`.

**Prerequisites:** Best done **after Phase 5** so all new screens (org onboarding, teams management) already exist and are translated once. Requires Phases 1‚Äì5 merged. If run earlier, the new-screen strings must be added when those phases land.

### Task 2: Generate ARB files from the existing maps (RO/HU/EN) with diacritics

**Files:**
- Create: `lib/l10n/app_ro.arb`
- Create: `lib/l10n/app_hu.arb`

- [ ] **Step 1: Extract the three maps into ARB JSON**

For every key in the existing `_ro`/`_hu`/`_en` maps in `lib/core/l10n/app_localizations.dart`, create a matching ARB entry. RO template gets metadata; HU/EN get values only. **While transcribing RO/HU, restore diacritics** (the maps have some but not all ‚Äî e.g. `Cine esti?` ‚Üí `Cine e»ôti?`). Example `app_ro.arb`:
```json
{
  "@@locale": "ro",
  "appName": "CampConnect",
  "roleSelectionTitle": "Cine e»ôti?",
  "imAGuide": "Sunt Ghid",
  "teamsCount": "{count, plural, one{{count} echipƒÉ} other{{count} echipe}}",
  "@teamsCount": {
    "placeholders": { "count": { "type": "int" } }
  },
  "confirmPointsMessage": "{action} {points} puncte pentru {team}?",
  "@confirmPointsMessage": {
    "placeholders": {
      "action": {"type": "String"},
      "points": {"type": "int"},
      "team": {"type": "String"}
    }
  }
}
```
Do the same for `app_hu.arb` (`"@@locale": "hu"`) and `app_en.arb` (`"@@locale": "en"`), values only (no `@`-metadata; the template defines shapes). Convert every hand-coded plural/parameterized method (`teamsCount`, `codesCount`, `confirmPointsMessage`, `generatedCodesFor`, `reassignKidsPrompt`, and the `relativeTime` pieces) to ICU messages with placeholders.

- [ ] **Step 2: Handle `relativeTime` as a set of ICU messages**

`relativeTime` builds "5 minutes ago" etc. Replace with four ICU keys:
```json
"justNow": "chiar acum",
"minutesAgo": "{count, plural, one{acum {count} minut} other{acum {count} minute}}",
"@minutesAgo": { "placeholders": { "count": {"type": "int"} } },
"hoursAgo": "{count, plural, one{acum {count} orƒÉ} other{acum {count} ore}}",
"@hoursAgo": { "placeholders": { "count": {"type": "int"} } },
"daysAgo": "{count, plural, one{acum {count} zi} other{acum {count} zile}}",
"@daysAgo": { "placeholders": { "count": {"type": "int"} } }
```
A small helper (Task 4) picks the right key from a `DateTime`.

- [ ] **Step 3: Generate and inspect**

Run:
```bash

```
Expected: `lib/l10n/app_localizations.g.dart` (class `AppL10n`) generated with a getter/method per key. Fix any ARB syntax errors it reports.

> **If the file lands in `.dart_tool/flutter_gen/gen_l10n/` instead of `lib/l10n/`:** your Flutter
> version still defaults to synthetic packages ‚Äî add `synthetic-package: false` to `l10n.yaml` and
> re-run. Recent Flutter (3.27+) writes into `arb-dir` by default and has deprecated that flag, so
> only add it if needed.

### Task 4: Migrate every call site from `AppLocalizations` to `AppL10n`

**Files:**
- Modify: every `.dart` under `lib/` that imports `core/l10n/app_localizations.dart`
- Create: `lib/core/utils/relative_time.dart`

- [ ] **Step 1: List all call sites**

Run:
```bash
grep -rln "AppLocalizations" lib
```
Record the file list (~30 files).

- [ ] **Step 2: Mechanical import + type swap (including the validators)**

In each file: replace `import 'package:camp_connect/core/l10n/app_localizations.dart';` (and relative variants) with `import 'package:camp_connect/l10n/app_localizations.g.dart';`, and replace `AppLocalizations.of(context)` with `AppL10n.of(context)`. The getter names are identical, so `l10n.someKey` mostly compiles unchanged. Parameterized calls change to the generated method signatures (same names: `l10n.teamsCount(n)`; note `confirmPointsMessage` becomes a 3-placeholder call `l10n.confirmPointsMessage(action, points, team)` ‚Äî its callers build `action` from the sign as before).

**In the same pass, retarget `lib/core/l10n/localized_validators.dart`:** change its field/constructor
type from `AppLocalizations` to `AppL10n` and swap its import. (Doing this here ‚Äî not later ‚Äî is what
lets Step 5's clean-analyze gate hold; the validators are consumed by the login screens you are
migrating in this very step.)

- [ ] **Step 3: Replace `relativeTime` usages with a helper**

Create `lib/core/utils/relative_time.dart`:
```dart
import 'package:camp_connect/l10n/app_localizations.g.dart';

String relativeTime(AppL10n l10n, DateTime dateTime) {
  final diff = DateTime.now().difference(dateTime);
  if (diff.inMinutes < 1) return l10n.justNow;
  if (diff.inMinutes < 60) return l10n.minutesAgo(diff.inMinutes);
  if (diff.inHours < 24) return l10n.hoursAgo(diff.inHours);
  return l10n.daysAgo(diff.inDays);
}
```
Replace every `l10n.relativeTime(x)` call with `relativeTime(l10n, x)` (import the helper). Grep to find them:
```bash
grep -rn "\.relativeTime(" lib
```

- [ ] **Step 4: Make all DateFormat calls locale-aware**

Run:
```bash
grep -rn "DateFormat(" lib
```
For each, pass the current locale, e.g. `DateFormat.yMMMMd(Localizations.localeOf(context).toString())` for long dates and `DateFormat.yMd(locale)` for short. Replace the hand-built US `'${date.month}/${date.day}/${date.year}'` in `guide_home_screen.dart:149` with `DateFormat.yMd(Localizations.localeOf(context).toString()).format(date)`. Ensure `intl`'s locale data is initialized ‚Äî add to `main.dart` before `runApp`: nothing extra needed for `DateFormat` with explicit locale, but if `LocaleDataException` appears, call `await initializeDateFormatting()` from `package:intl/date_symbol_data_local.dart`.
