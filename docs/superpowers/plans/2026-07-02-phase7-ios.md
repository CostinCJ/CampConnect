# Phase 7 — iOS Enablement (build from Windows, no Mac) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make CampConnect build, sign, and run on iOS with zero Mac access — all code/config from Windows, all compilation/signing on a cloud macOS CI (Codemagic), tested via TestFlight on one physical iPhone. Fix the three iOS-specific blockers found in the sweep and the OSM tile-policy issue.

**Architecture:** Two tracks. **(A) Code/config** (done on Windows, in this repo): add iOS privacy purpose strings, make journal export use `Printing.sharePdf` on iOS, downgrade the APNs `critical` payload to `time-sensitive`, and swap the map tile provider to a keyed service. **(B) Operational** (accounts + cloud CI, not code): Apple Developer Program, a Codemagic `codemagic.yaml` workflow with App Store Connect API automatic signing, and a used iPhone for TestFlight testing.

**Tech Stack:** Flutter iOS, `Info.plist`, `printing`, MapTiler/Stadia tiles, Codemagic CI, App Store Connect API, TestFlight.

**Branch:** `phase7-ios`.

**Prerequisites:** **Phase 1 (LLM removal) is mandatory** — the `fllama` git dependency breaks cloud iOS builds. Phases 2–6 recommended but the iOS config fixes are independent. You will need: an Apple Developer account ($99/yr), a Codemagic account (free tier), and a physical iPhone.

### Task 2: Make journal export work on iOS

**Files:**
- Modify: `lib/features/journal/presentation/journal_export_screen.dart`

**Root cause:** `FileSaverService.saveToDownloads` is implemented only in the Android `MainActivity` MethodChannel; on iOS the call throws `MissingPluginException`. iOS has no user-facing Downloads folder anyway — the correct pattern is the share sheet.

- [ ] **Step 1: Branch the save path by platform**

In `journal_export_screen.dart`, import `dart:io`:
```dart
import 'dart:io' show Platform;
import 'package:printing/printing.dart';
```
In `_generateAndSave`, replace the unconditional:
```dart
      // Save directly to Downloads folder
      await FileSaverService.saveToDownloads(
        bytes: bytes,
        filename: filename,
      );
```
With:
```dart
      if (Platform.isAndroid) {
        await FileSaverService.saveToDownloads(bytes: bytes, filename: filename);
      } else {
        // iOS has no Downloads folder — present the system share sheet instead.
        await Printing.sharePdf(bytes: bytes, filename: filename);
      }
```

- [ ] **Step 2: Adjust the success banner copy for iOS**

The banner currently hard-codes `Downloads/$_filename`. Make it conditional so iOS doesn't claim a Downloads path:
```dart
                  child: Text(
                    Platform.isAndroid
                        ? '${l10n.pdfExported}\nDownloads/$_filename'
                        : l10n.pdfExported,
```

- [ ] **Step 3: Analyze**

Run:
```bash
flutter analyze lib/features/journal/presentation/journal_export_screen.dart
```
Expected: no issues.

- [ ] **Step 4: Commit**

```bash
git add lib/features/journal/presentation/journal_export_screen.dart
git commit -m "fix(ios): use share sheet for journal PDF export on iOS"
```

### Task 3: Downgrade emergency push from `critical` to `time-sensitive`

**Files:**
- Modify: `functions/index.js`
- Modify: `lib/shared/services/fcm_service.dart`

**Root cause:** APNs `interruption-level: critical` and `criticalAlert: true` require a special Apple entitlement you will not have; the push gets downgraded/rejected on iOS.

- [ ] **Step 1: Change the APNs payload in the emergency function**

In `functions/index.js` `onEmergencyAlertCreated`, replace:
```javascript
      apns: {
        payload: {
          aps: {
            sound: "default",
            "interruption-level": "critical",
          },
        },
      },
```
With:
```javascript
      apns: {
        payload: {
          aps: {
            sound: "default",
            "interruption-level": "time-sensitive",
          },
        },
      },
```

- [ ] **Step 2: Remove `criticalAlert` from the permission request**

In `fcm_service.dart` `requestPermission`, remove `criticalAlert: true,` from the `requestPermission` call (leave `alert`, `badge`, `sound`).

- [ ] **Step 3: Lint + analyze**

Run:
```bash
cd /d/CampConnect/functions && npx eslint index.js || true
cd /d/CampConnect && flutter analyze lib/shared/services/fcm_service.dart
```
Expected: no fatal errors.

- [ ] **Step 4: Commit**

```bash

```

### Task 4: Swap map tiles off the OSM public server

**Files:**
- Modify: `lib/features/map/presentation/map_screen.dart`
- Modify: `lib/core/constants/app_constants.dart`

**Root cause:** `tile.openstreetmap.org` (`map_screen.dart:112`) forbids production app distribution + bulk caching (which FMTC does). A keyed provider is required for store distribution.

- [ ] **Step 1: Add a tile config constant**

Create a MapTiler (or Stadia) account (free tier) and get an API key. In `app_constants.dart` add:
```dart
  // Map tiles — keyed provider (OSM public tiles forbid app distribution + caching).
  // Provide via --dart-define=MAPTILER_KEY=xxxx so the key is not committed.
  // The /256/ path serves 256px tiles, matching flutter_map's default tile size.
  static const String maptilerKey =
      String.fromEnvironment('MAPTILER_KEY', defaultValue: '');
  static const String tileUrlTemplate =
      'https://api.maptiler.com/maps/outdoor-v2/256/{z}/{x}/{y}.png?key=$maptilerKey';
```

- [ ] **Step 2: Use it in the TileLayer + visible attribution**

In `map_screen.dart`, replace:
```dart
                urlTemplate: 'https://tile.openstreetmap.org/{z}/{x}/{y}.png',
                userAgentPackageName: 'com.campconnect.app',
```
With:
```dart
                urlTemplate: AppConstants.tileUrlTemplate,
                userAgentPackageName: 'com.campconnect.app',
```
(`additionalOptions` is only for URL-template placeholders — attribution is a separate layer.)
MapTiler and OSM both require **visible attribution**: add a `RichAttributionWidget` as the last
entry in the `FlutterMap` `children:` list:
```dart
              RichAttributionWidget(
                attributions: [
                  TextSourceAttribution('© MapTiler © OpenStreetMap contributors'),
                ],
              ),
```

- [ ] **Step 3: Pass the key in build/run commands**

Document that dev/CI builds must pass `--dart-define=MAPTILER_KEY=<key>`. Add it to the Codemagic env (Task 6) as an encrypted variable.

- [ ] **Step 4: Analyze**

Run:
```bash
flutter analyze lib/features/map/presentation/map_screen.dart
```
Expected: no issues.

- [ ] **Step 5: Commit**

```bash
git add lib/features/map/presentation/map_screen.dart lib/core/constants/app_constants.dart
git commit -m "fix: use keyed MapTiler tiles with attribution instead of OSM public server"
```

### Task 5: iOS build config prerequisites (Windows-editable)

**Files:**
- Modify: `ios/Runner.xcodeproj/project.pbxproj` (bundle id / deployment target — via Flutter defaults where possible)
- Modify: `ios/Podfile` (platform version)

- [ ] **Step 1: Set the iOS deployment target**

In `ios/Podfile`, ensure the top line is uncommented and set:
```ruby
platform :ios, '13.0'
```
(Firebase + flutter_local_notifications require iOS 13+.)

- [ ] **Step 2: Confirm the bundle identifier**

The Android `applicationId` is `com.campconnect.camp_connect`. Confirm the iOS bundle id matches what you will register in the Apple Developer portal. It is set in `ios/Runner.xcodeproj/project.pbxproj` as `PRODUCT_BUNDLE_IDENTIFIER`. Record the exact value — you register this exact id as an App ID in Task 6.

- [ ] **Step 3: Confirm the GoogleService-Info.plist is present**

Run:
```bash
ls ios/Runner/GoogleService-Info.plist
```
Expected: exists (it does). For the dev flavor from Phase 5, you also need the dev project's `GoogleService-Info.plist` — keep prod for the store build.

- [ ] **Step 4: Push-notification entitlements (required for FCM on iOS)**

FCM on iOS needs three things that Xcode normally sets up — all doable without a Mac:

**(a) Create `ios/Runner/Runner.entitlements`:**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
	<key>aps-environment</key>
	<string>production</string>
</dict>
</plist>
```

**(b) Reference it from the Xcode project.** In `ios/Runner.xcodeproj/project.pbxproj`, add
`CODE_SIGN_ENTITLEMENTS = Runner/Runner.entitlements;` to the build settings of each Runner
configuration (Debug, Release, Profile) — place it alongside the existing
`ASSETCATALOG_COMPILER_APPICON_NAME` line in each `buildSettings` block, and add the file itself to
the `PBXFileReference`/group sections mirroring how `Info.plist` is referenced. Hand-editing pbxproj
is the fiddliest step of this phase — if the Codemagic build then fails on entitlements, the
fallback is one hour on a rented cloud Mac (MacinCloud) to click the "Push Notifications" capability
in Xcode, then commit the resulting diff.

**(c) Portal + Firebase config (browser-only):** in the Apple Developer portal, enable the **Push
Notifications** capability on the App ID; create an **APNs Auth Key** (`.p8`, Keys section) and
upload it (with Key ID + Team ID) to **Firebase console → Project settings → Cloud Messaging →
Apple app configuration**. Without this, FCM sends will silently not arrive on iOS.

- [ ] **Step 5: Commit**

```bash
git add ios/Podfile ios/Runner/Runner.entitlements ios/Runner.xcodeproj/project.pbxproj
git commit -m "chore(ios): deployment target, push entitlements, and APNs wiring"
```

### Task 6: Codemagic CI — build + sign + TestFlight (operational, no Mac)

**Files:**
- Create: `codemagic.yaml`

- [ ] **Step 1: Apple Developer Program enrollment (manual)**

Enroll at developer.apple.com ($99/yr). In App Store Connect, create the app record with the exact bundle id from Task 5. Generate an **App Store Connect API key** (Users and Access → Integrations → keys): note the Key ID, Issuer ID, and download the `.p8` — these let Codemagic sign without any Mac/Keychain.

- [ ] **Step 2: Create the Codemagic workflow**

Create `codemagic.yaml`:
```yaml
workflows:
  ios-testflight:
    name: iOS TestFlight
    instance_type: mac_mini_m2
    max_build_duration: 60
    integrations:
      app_store_connect: CampConnect ASC API Key
    environment:
      groups:
        - campconnect_secrets   # holds MAPTILER_KEY
      ios_signing:
        distribution_type: app_store
        bundle_identifier: com.campconnect.camp_connect
      flutter: stable
      xcode: latest
      cocoapods: default
    scripts:
      - name: Flutter pub get
        script: flutter pub get
      - name: Generate l10n
        script: flutter gen-l10n
      - name: Install pods
        script: |
          find . -name "Podfile" -execdir pod install \;
      - name: Apply signing profiles
        script: |
          # Writes /Users/builder/export_options.plist and injects the
          # cloud-generated certificate + provisioning profile into the project.
          xcode-project use-profiles
      - name: Build IPA
        script: |
          flutter build ipa --release \
            --dart-define=MAPTILER_KEY=$MAPTILER_KEY \
            --export-options-plist=/Users/builder/export_options.plist
    artifacts:
      - build/ios/ipa/*.ipa
    publishing:
      app_store_connect:
        auth: integration
        submit_to_testflight: true
```
> The `app_store_connect` integration name (`CampConnect ASC API Key`) is created once in the Codemagic
> UI from the `.p8`/Key ID/Issuer ID. `ios_signing.distribution_type: app_store` enables **automatic**
> certificate + provisioning-profile generation in the cloud — no Mac, no Keychain.

- [ ] **Step 3: Set encrypted env vars in Codemagic**

In the Codemagic UI, create the `campconnect_secrets` group with `MAPTILER_KEY` (encrypted). Connect the GitHub repo.

- [ ] **Step 4: Trigger the first build (manual)**

Push the branch to GitHub (with user authorization) and start the `ios-testflight` workflow in Codemagic. Expected outcome: a green build producing a signed `.ipa` uploaded to TestFlight. Debug any pod/signing errors from the Codemagic log (common first-build fixes: bundle-id mismatch, missing dev-team — the ASC integration resolves the team automatically).

- [ ] **Step 5: Commit the workflow file**

```bash
git add codemagic.yaml
git commit -m "ci: add Codemagic iOS workflow with automatic signing and TestFlight upload"
```
